Ensure all documentation properly credits sources of inspiration, borrowed content, and external resources.

## What This Command Does

1. **Scans documentation** for references to external sources
2. **Identifies missing attributions** for:
    - Code or configurations from external sources
    - Concepts or methodologies from blog posts/articles
    - Open source projects or frameworks
    - Community-contributed content
    - Adapted or modified work
3. **Suggests attribution format** with required elements
4. **Adds Attribution & Credits section** if missing

## Attribution Requirements

Every external source should be credited with:

- **Source Name/Title**: Clear identification
- **Source URL**: Direct link to original
- **Author/Organization**: Creator or maintainer
- **License** (if applicable): Open source license
- **Usage Description**: How it influenced the work

## Example Attribution

```markdown
## Attribution & Credits

This project was inspired by and built upon the following resources:

### Resource Name
- **Source**: https://example.com/resource
- **Author**: Jane Developer
- **License**: MIT
- **Usage**: Core algorithm adapted from this implementation
```

## When Attribution is Required

- Using external code, configs, or patterns
- Adapting existing work
- Drawing inspiration from published sources
- Following methodologies from articles/blogs
- Using open source projects
- Incorporating community contributions

## Best Practices

- **Be generous** - When in doubt, attribute
- **Be specific** - Explain how the source was used
- **Include licenses** - Respect open source terms
- **Link directly** - Provide accurate URLs
- **Stay accurate** - Verify author names and sources

Use this command to ensure proper intellectual property respect and transparency in all documentation.
